## Code Changes

### Enhance Traps
- Increase time-waste potential creatively
- Add layers (fake files, escalating nonsense, loops)
- Make detection realistic but ultimately futile

### Safety Gates
- **NEVER** modify whitelist without explicit approval
- **ALWAYS** test in non-production environment first
- **ALWAYS** whitelist yourself before testing entry logic
- **DOCUMENT** new traps in ANTI_SNOOP_README.md

### Testing Philosophy
- Test traps as a snooper would encounter them
- Verify whitelist allows team members to bypass
- Check logging captures the full "performance"
- Confirm no data leaks in fake files (they're honeypot, not real)
